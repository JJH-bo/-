# V6 城市建模规格书｜张宇第15讲《微分方程》知识城市

## 一、总定位

V6 的目标不是继续加节点、连线、标签，而是重建城市规划。

**城市第一，知识第二层嵌入。**

用户进入以后，先感到：

> 我进入了一座“微分方程城市”。

然后再通过城区、建筑、道路、桥梁、海湾、地铁线，理解张宇第15讲的知识结构。

## 二、设计总句

微分方程不是一张知识网，而是一座按重要性、章节层级、方法关系、学习路径规划出来的城市。最难、最重要的知识板块要占据城市最核心、最高、最有地标感的区域。

## 三、城市主结构

城市整体采用：

```text
微分河 + 五大城区 + 曼哈顿核心岛 + 欧拉跨江大桥 + 应用海湾 + 风险训练场
```

地理元素不是装饰，每个元素都有数学含义。

### 1. 微分河

**微分河** 表示变量随过程流动，也表示微分方程的核心是“变化率关系”。

两岸划分：

- 左岸：概念与一阶方程。
- 中央岛：高阶线性核心区。
- 右岸/海湾：欧拉转化与应用建模。

### 2. 主路系统

一条主路从入口港出发：

```text
概念 → 一阶方程 → 可降阶 → 高阶线性 → 欧拉 → 应用
```

这条路叫 **张宇主线大道**。

### 3. 桥梁系统

桥梁只用于关键结构转化：

- 伯努利 → 一阶线性。
- 可降阶 → 一阶方程。
- 欧拉 → 常系数线性。
- 现实关系 → 微分方程。

## 四、城区规划

### 1. 入口港·概念广场

定位：低矮、开阔、清晰的城市入口。

建筑：

- 微分方程纪念碑。
- 阶塔。
- 解/通解/特解三联馆。
- 初始条件检查站。

数学职责：

- 微分方程及其阶。
- 常微分方程。
- 线性微分方程。
- 微分方程的解、通解、特解。
- 初始条件与特解。

### 2. 皇后·一阶方程区

定位：基础大区，面积大、街区多、道路清晰。

建筑/街区：

- 可分离变量街。
- 齐次斜率坡区。
- 一阶线性馆。
- 伯努利转换门。
- 全微分庭院。

视觉风格：

- 中等高度。
- 街区网格明显。
- 每条方法街有自己的入口。
- 建筑比曼哈顿低，但数量更多。

数学职责：

- 直接可分离。
- 换元后可分离。
- 齐次型微分方程。
- 一阶线性微分方程。
- 伯努利方程。
- 二阶可降阶方程转一阶。
- 全微分方程。

### 3. 布鲁克林·可降阶工业区

定位：工业区、加工区、转换区。

建筑：

- 可降阶总电梯。
- 缺 `y` 厂房。
- 缺 `x` 链式车间。
- 变量角色塔。
- 二阶下降平台。

数学职责：

- 缺 `y`：令 `p=y'`，把 `y''` 变成 `p'`。
- 缺 `x`：令 `p=p(y)`，把 `y''` 变成 `p dp/dy`。
- 降阶不是套路，而是换研究对象。

### 4. 曼哈顿·高阶线性核心岛

定位：城市核心，最高、最重要、最难。

建筑：

- 线性解结构大楼。
- 特征方程中央塔。
- 不等实根双塔。
- 重根摩天楼。
- 复根旋转塔。
- 非齐次特解工厂。
- 待定系数实验楼。
- 欧拉桥入口。

视觉风格：

- 高密度。
- 强天际线。
- 地标明显。
- 从远处一眼看出“这里是核心”。

数学职责：

- 高阶线性解结构。
- 二阶常系数齐次线性方程。
- 特征方程。
- 不等实根、重根、复根。
- 非齐次通解 = 齐次通解 + 特解。

### 5. 欧拉跨江大桥

定位：连接特殊结构与常系数线性区。

桥的一端：

```text
x^2y'' + pxy' + qy = f(x)
```

桥中间：

```text
x = e^t
```

桥另一端接入曼哈顿高阶线性区。

数学职责：

欧拉方程不是孤立方法，而是通过变量变换进入常系数线性体系。

### 6. 应用海湾·现实建模码头

定位：现实世界接口。

场景：

- 几何应用码头。
- 自行车轨迹馆。
- 飞机减速跑道。
- 牛顿冷却实验塔。

数学职责：

应用题不是套公式，而是把现实关系翻译成变化率关系。

### 7. 红钩·风险训练场

定位：城市里的危险训练区。

风险建筑：

- 除零丢解坑。
- 绝对值门。
- 常数合并站。
- 变量角色塔。
- 初值定符号检查点。

数学职责：

错误不是附属内容，而是学习系统中的独立区域。

### 8. 灰岛·差分方程选学区

定位：远处灰色岛屿。

因为张宇第15讲中差分方程只对数学三要求，数学一不作为当前主线，所以它保留为远景灰岛，不干扰主城。

## 五、城市优先级与高度规则

### 1. 城市核心规则

越重要、越难、越综合，越靠近城市中心，越像曼哈顿。

排序：

1. 曼哈顿·高阶线性区。
2. 皇后·一阶方程区。
3. 布鲁克林·可降阶工业区。
4. 应用海湾。
5. 入口港。
6. 风险训练场。
7. 灰岛。

### 2. 建筑高度规则

高度表达“抽象层级”：

- 概念区：低。
- 一阶方程区：中。
- 可降阶区：中高。
- 高阶线性区：最高。
- 应用区：不靠高度，靠场景辨识。
- 风险区：低矮但颜色强烈。

### 3. 地标夸张规则

地标建筑要夸张：

- 特征方程塔最高。
- 一阶线性馆明显。
- 伯努利转换门有巨大入口。
- 可降阶电梯高。
- 欧拉桥长。
- 冷却实验塔有曲线。
- 自行车馆有轨迹线。

## 六、标签系统

标签按距离变化：

### 远距离

显示：

- 城区名。
- 重要地标名。
- 主路线名。

### 中距离

显示：

- 建筑名。
- 极短公式。

### 近距离

漂浮文字消失，改为：

- 建筑牌匾。
- 门口公式牌。
- 点击后侧边面板。

## 七、昼夜模式

### 1. 白天城市模式

用于学习和识别：

- 白底低饱和。
- 蓝绿色水体。
- 灰白建筑。
- 重要建筑轻微强调。
- 标签清晰。
- 公式少。

### 2. 知识夜景模式

用于看关系和结构：

- 城市变暗。
- 地铁线、主路、桥梁发光。
- 重要建筑亮灯。
- 公式牌发光。
- 后面可接入个人掌握状态。

## 八、V6 硬验收标准

1. 有真实城区布局，不是一堆盒子。
2. 有道路、桥梁、河流、广场。
3. 天际线有层次，远看能分区。
4. 材质和光影明显提升。
5. 能让人有“进入微分方程城市”的感觉。

## 九、下一阶段开发策略

V6 之后的开发顺序：

1. 城市形态继续细化。
2. 城区身份继续增强。
3. 地标建筑的数学机制可视化。
4. 道路、桥梁、地铁线变成真正的知识关系基础设施。
5. 最后再回到建筑内部学习。
